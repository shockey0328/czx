# 橙子学数据看板 - 使用指南

## 快速开始

### 方法一：一键启动（Windows）

双击运行 `启动看板.bat` 文件，脚本会自动：
1. 启动主看板服务器（端口8000）
2. 启动月度核心数据看板React服务（端口3000）

启动完成后，在浏览器中访问：http://localhost:8000

### 方法二：手动启动

#### 1. 启动主看板

在项目根目录打开终端，运行以下命令之一：

```bash
# 使用Python（推荐）
python -m http.server 8000

# 或使用Node.js
npx http-server -p 8000
```

#### 2. 启动月度核心数据看板（可选）

如果需要查看月度核心数据，需要单独启动React应用：

```bash
# 进入目录
cd 核心数据看板（月度）

# 首次运行需要安装依赖
npm install

# 启动开发服务器
npm start
```

## 功能说明

### 1. 周度/月度切换

点击顶部的"周度"或"月度"按钮，可以切换不同的时间维度：

- **周度**：查看周度核心数据和搜索数据
- **月度**：查看月度核心数据和各模块渗透率

### 2. 看板类型选择

使用右侧的下拉菜单选择具体的看板类型：

#### 周度看板
- **核心数据**：周度核心数据指标、趋势分析、B端数据、AI分析
- **搜索数据**：热搜词分析、搜索行为漏斗、转化趋势、功能留存

#### 月度看板
- **核心数据**：月度核心指标、趋势图表、AI智能分析（需要先启动React服务）
- **各模块渗透率**：模块渗透率趋势、排名、热力图、数据表格

### 3. 看板交互

每个看板都保留了原有的所有交互功能：
- 图表缩放和筛选
- 数据导出
- 时间范围选择
- AI分析功能
- 等等

## 常见问题

### Q1: 月度核心数据看板显示"需要运行React服务器"

**解决方法：**
1. 打开新的终端窗口
2. 执行：`cd 核心数据看板（月度）`
3. 执行：`npm install`（首次运行）
4. 执行：`npm start`
5. 等待服务启动后，刷新主看板页面

### Q2: 看板加载失败或显示空白

**可能原因：**
- 文件路径不正确
- 浏览器缓存问题
- 本地服务器未启动

**解决方法：**
1. 确保在项目根目录启动服务器
2. 清除浏览器缓存后重试
3. 检查浏览器控制台的错误信息
4. 尝试使用其他浏览器

### Q3: 启动脚本无法运行

**解决方法：**
1. 确保已安装Python或Node.js
2. 确保已安装npm（Node.js自带）
3. 以管理员身份运行脚本
4. 手动按照"方法二"启动

### Q4: 端口被占用

**解决方法：**
```bash
# 查看端口占用（Windows）
netstat -ano | findstr :8000
netstat -ano | findstr :3000

# 结束占用端口的进程
taskkill /PID <进程ID> /F

# 或者使用其他端口启动
python -m http.server 8080
```

## 性能优化建议

1. 关闭不需要的浏览器标签页
2. 定期清理浏览器缓存
3. 确保网络连接稳定（部分看板使用CDN资源）
4. 使用性能较好的设备查看复杂图表

## 数据更新

各看板的数据文件位置：
- 周度核心数据：`核心数据看板（周度）/*.csv`
- 搜索数据：`搜索数据看板（周度）/*.csv`
- 月度核心数据：`核心数据看板（月度）/public/*.csv`
- 各模块渗透率：`各模块渗透率看板（月度）/*.csv`

更新数据后，刷新页面即可看到最新数据。
