## 📝 需要你完成的步骤

### 步骤 1：更新远程仓库地址

打开命令行（PowerShell 或 CMD），在项目目录执行：

```bash
# 移除当前的远程仓库
git remote remove origin

# 添加你的 GitHub 仓库（替换 YOUR_USERNAME 为你的 GitHub 用户名）
git remote add origin https://github.com/YOUR_USERNAME/czx-dashboard.git
```

### 步骤 2：推送到 GitHub

```bash
# 推送到 GitHub
git push -u origin main
```

如果提示需要认证，可以：
1. 使用 GitHub Desktop（推荐）
2. 配置 Personal Access Token
3. 使用 SSH 密钥

### 步骤 3：部署到 Vercel

#### 方法 A：通过 Vercel 网站（最简单）

1. 访问 [https://vercel.com](https://vercel.com)
2. 使用 GitHub 账号登录
3. 点击 "Add New..." → "Project"
4. 选择 `czx-dashboard` 仓库
5. 保持默认配置：
   - Framework Preset: Other
   - Root Directory: ./
   - Build Command: (留空)
   - Output Directory: (留空)
6. 点击 "Deploy"
7. 等待 1-2 分钟完成部署

#### 方法 B：通过 Vercel CLI

```bash
# 1. 安装 Vercel CLI
npm install -g vercel

# 2. 登录
vercel login

# 3. 部署
vercel

# 4. 生产环境部署
vercel --prod
```

## 🔧 如果遇到网络问题

### 使用 GitHub Desktop（推荐）

1. 下载并安装 [GitHub Desktop](https://desktop.github.com/)
2. 打开 GitHub Desktop
3. File → Add Local Repository
4. 选择项目文件夹
5. 点击 "Publish repository"
6. 输入仓库名：`czx-dashboard`
7. 点击 "Publish repository"

### 配置代理（如果需要）

```bash
# 设置 Git 代理
git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy http://127.0.0.1:7890

# 取消代理
git config --global --unset http.proxy
git config --global --unset https.proxy
```

## 📞 需要帮助？

如果遇到问题：

1. **GitHub 推送失败**
   - 检查仓库是否存在
   - 确认有推送权限
   - 尝试使用 GitHub Desktop

2. **Vercel 部署失败**
   - 查看部署日志
   - 检查 vercel.json 配置
   - 确认文件路径正确

3. **页面显示异常**
   - 清除浏览器缓存
   - 检查控制台错误
   - 验证资源文件路径
