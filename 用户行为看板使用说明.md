# 用户行为看板使用说明

## 为什么需要启动服务器？

用户行为看板使用了以下技术，必须通过Node.js服务器运行：

1. **DeepSeek AI分析** - 需要后端保护API密钥安全
2. **大数据处理** - 支持百万级用户行为日志查询
3. **高性能数据库** - 使用分片存储和内存索引
4. **跨域请求处理** - 需要服务器端CORS支持

## 快速启动（3步）

### 方法一：使用启动脚本（推荐）

1. 双击运行 `启动用户行为看板.bat`
2. 等待服务器启动（首次运行会自动安装依赖）
3. 在主看板中选择"用户行为"

### 方法二：手动启动

1. 打开终端/命令行
2. 执行以下命令：
```bash
cd 用户行为看板（周度）
npm install  # 首次运行
node server-with-db.js
```
3. 看到 "服务器运行在 http://localhost:3001" 后，在主看板中选择"用户行为"

## 访问方式

启动服务器后，可以通过以下方式访问：

1. **通过主看板**：在主看板选择"周度" → "用户行为"
2. **直接访问**：浏览器打开 http://localhost:3001/dashboard-db.html

## 功能特点

- ✅ 时间段筛选（起始日期和结束日期）
- ✅ 多用户ID查询（逗号分隔）
- ✅ DeepSeek AI智能分析用户行为
- ✅ 对话式交互界面
- ✅ 支持标准分析和针对性问题分析
- ✅ 高性能数据库（支持百万级记录）

## 数据准备

将Excel文件（格式：`2026年3月3日用户行为日志.xlsx`）放在 `用户行为看板（周度）` 目录下，然后运行：

```bash
cd 用户行为看板（周度）
node db-manager.js init
```

## 常见问题

### Q: 为什么不能像其他看板一样直接打开？
A: 用户行为看板需要AI分析和大数据处理能力，必须通过服务器运行以保证安全性和性能。

### Q: 服务器启动失败怎么办？
A: 
1. 确保已安装Node.js（版本 >= 14）
2. 检查3001端口是否被占用
3. 查看终端错误信息

### Q: 可以修改端口吗？
A: 可以，编辑 `server-with-db.js` 文件，修改 `PORT` 变量，同时需要更新 `main.js` 中的配置。

### Q: 数据更新后需要重启服务器吗？
A: 需要。运行 `node db-manager.js rebuild` 重建索引，然后重启服务器。
