# 解决 GitHub 连接问题

## 问题原因
无法连接到 GitHub 服务器，可能的原因：
1. 网络问题或防火墙
2. 需要配置代理
3. DNS 解析问题

## 解决方案

### 方案一：使用 SSH 代替 HTTPS（推荐）

#### 1. 生成 SSH 密钥（如果还没有）
```bash
ssh-keygen -t ed25519 -C "your_email@example.com"
```
按回车使用默认路径，可以设置密码或直接回车跳过

#### 2. 查看公钥
```bash
cat ~/.ssh/id_ed25519.pub
```
或在 Windows 上：
```bash
type C:\Users\你的用户名\.ssh\id_ed25519.pub
```

#### 3. 添加 SSH 密钥到 GitHub
1. 复制上一步显示的公钥内容
2. 访问 https://github.com/settings/keys
3. 点击 "New SSH key"
4. 粘贴公钥，点击 "Add SSH key"

#### 4. 使用 SSH 地址推送
```bash
git remote add origin git@github.com:你的用户名/search-dashboard.git
git branch -M main
git push -u origin main
```

### 方案二：配置代理（如果你使用代理）

#### 如果使用 HTTP 代理
```bash
git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy http://127.0.0.1:7890
```

#### 如果使用 SOCKS5 代理
```bash
git config --global http.proxy socks5://127.0.0.1:7890
git config --global https.proxy socks5://127.0.0.1:7890
```

#### 取消代理设置
```bash
git config --global --unset http.proxy
git config --global --unset https.proxy
```

### 方案三：修改 hosts 文件

#### Windows
1. 以管理员身份打开记事本
2. 打开文件：`C:\Windows\System32\drivers\etc\hosts`
3. 添加以下内容：
```
140.82.113.4 github.com
199.232.69.194 github.global.ssl.fastly.net
```
4. 保存文件

#### 然后重试推送
```bash
git remote add origin https://github.com/你的用户名/search-dashboard.git
git branch -M main
git push -u origin main
```

### 方案四：使用 GitHub Desktop（最简单）

1. 下载安装 GitHub Desktop：https://desktop.github.com/
2. 登录 GitHub 账号
3. 点击 "Add" → "Add existing repository"
4. 选择当前项目文件夹
5. 点击 "Publish repository"
6. 填写仓库名称，点击 "Publish repository"

### 方案五：直接使用 Vercel CLI（跳过 GitHub）

如果 GitHub 连接一直有问题，可以直接部署到 Vercel：

```bash
# 1. 安装 Vercel CLI
npm install -g vercel

# 2. 登录（会打开浏览器）
vercel login

# 3. 部署
vercel

# 4. 生产环境部署
vercel --prod
```

这样可以跳过 GitHub，直接从本地部署到 Vercel。

## 测试连接

### 测试 HTTPS 连接
```bash
git ls-remote https://github.com/你的用户名/search-dashboard.git
```

### 测试 SSH 连接
```bash
ssh -T git@github.com
```

成功会显示：`Hi 你的用户名! You've successfully authenticated...`

## 推荐方案

1. **最简单**：使用 GitHub Desktop（方案四）
2. **最稳定**：配置 SSH（方案一）
3. **最快速**：直接用 Vercel CLI（方案五）

## 需要帮助？

如果以上方案都不行，可以：
1. 检查网络连接
2. 尝试使用手机热点
3. 联系网络管理员
4. 使用 VPN 或代理
