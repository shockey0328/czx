# 重新部署到 Vercel

## 问题
部署后显示 404 错误，这是因为 Vercel 配置问题。

## 已修复
✅ 已更新 vercel.json 配置文件
✅ 已提交到 Git

## 重新部署方法

### 方法一：通过 Vercel Dashboard（推荐）

1. 访问 https://vercel.com/dashboard
2. 找到你的项目
3. 点击项目名称进入详情页
4. 点击右上角的 "..." 菜单
5. 选择 "Redeploy"
6. 点击 "Redeploy" 确认

### 方法二：通过 Vercel CLI

如果已安装 Vercel CLI：
```bash
vercel --prod
```

如果未安装，先安装：
```bash
npm install -g vercel
vercel login
vercel --prod
```

### 方法三：删除项目重新部署

1. 在 Vercel Dashboard 中删除当前项目
2. 重新运行：
```bash
vercel --prod
```

## 验证部署

部署成功后，访问你的 Vercel 链接，应该能看到完整的看板页面。

## 如果还是 404

### 检查项目根目录
确保以下文件在项目根目录：
- ✅ index.html
- ✅ app.js
- ✅ logo.png
- ✅ 所有 CSV 文件
- ✅ vercel.json

### 检查 vercel.json
确保内容为：
```json
{
  "rewrites": [
    {
      "source": "/",
      "destination": "/index.html"
    }
  ]
}
```

### 查看部署日志
1. 在 Vercel Dashboard 中点击项目
2. 点击最新的部署
3. 查看 "Build Logs" 和 "Function Logs"
4. 查找错误信息

## 常见问题

### Q: 部署成功但页面空白
A: 检查浏览器控制台（F12），查看是否有 JavaScript 错误

### Q: CSV 文件加载失败
A: 确保 CSV 文件都是 UTF-8 编码

### Q: 图表不显示
A: 检查网络连接，ECharts 通过 CDN 加载
