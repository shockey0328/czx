# 用户行为看板线上部署指南

## 方案选择

用户行为看板需要Node.js服务器支持，有以下几种部署方案：

### 方案A：Railway（推荐 - 最简单）

**优点：**
- 免费部署Node.js应用
- 自动从GitHub部署
- 提供免费数据库
- 配置简单

**步骤：**

1. **注册Railway账号**
   - 访问：https://railway.app
   - 使用GitHub账号登录

2. **创建新项目**
   - 点击 "New Project"
   - 选择 "Deploy from GitHub repo"
   - 选择你的仓库：`shockey0328/czx`

3. **配置部署**
   - 选择 `用户行为看板（周度）` 目录
   - Railway会自动检测到Node.js项目
   - 设置环境变量（如果需要）：
     ```
     PORT=3001
     DEEPSEEK_API_KEY=你的API密钥
     ```

4. **部署**
   - 点击 "Deploy"
   - 等待部署完成（约2-3分钟）
   - Railway会提供一个公网URL，如：`https://your-app.railway.app`

5. **更新主看板配置**
   - 修改 `main.js` 中的用户行为看板URL
   - 将 `http://localhost:3001` 改为Railway提供的URL

**费用：**
- 免费额度：每月500小时运行时间
- 足够个人和小团队使用

---

### 方案B：Render

**优点：**
- 完全免费（有限制）
- 自动从GitHub部署
- 配置简单

**步骤：**

1. **注册Render账号**
   - 访问：https://render.com
   - 使用GitHub账号登录

2. **创建Web Service**
   - 点击 "New +" → "Web Service"
   - 连接GitHub仓库
   - 选择 `用户行为看板（周度）` 目录

3. **配置服务**
   ```
   Name: czx-user-behavior
   Environment: Node
   Build Command: npm install
   Start Command: node server-with-db.js
   ```

4. **设置环境变量**
   ```
   PORT=10000
   DEEPSEEK_API_KEY=你的API密钥
   ```

5. **部署**
   - 点击 "Create Web Service"
   - 等待部署完成
   - Render会提供URL：`https://czx-user-behavior.onrender.com`

**限制：**
- 免费版15分钟无活动会休眠
- 首次访问需要等待唤醒（约30秒）

---

### 方案C：Vercel Serverless Functions（需要改造）

**优点：**
- 与现有项目在同一平台
- 免费额度充足

**缺点：**
- 需要将代码改造为Serverless架构
- 数据库需要使用外部服务（如MongoDB Atlas）
- 改造工作量较大

**是否推荐：** 不推荐，除非你熟悉Serverless架构

---

## 推荐方案：Railway

**总结：**
1. Railway最简单，直接部署Node.js应用
2. 免费额度充足
3. 自动从GitHub部署，推送代码即更新
4. 提供HTTPS和自定义域名

**部署后的架构：**
```
用户浏览器
    ↓
Vercel (主看板 - 静态)
    ↓
Railway (用户行为看板 - Node.js服务器)
    ↓
DeepSeek AI API
```

---

## 部署后需要做的事

1. **获取Railway URL**
   - 部署完成后，Railway会提供一个URL
   - 例如：`https://czx-user-behavior-production.up.railway.app`

2. **更新main.js配置**
   ```javascript
   'user-behavior-weekly': {
       name: '用户行为',
       path: 'https://你的railway域名/dashboard-db.html',
       type: 'server',
       serverCommand: 'node server-with-db.js',
       serverPath: '用户行为看板（周度）'
   }
   ```

3. **推送到GitHub**
   - Railway会自动检测更新并重新部署

4. **测试**
   - 访问主看板
   - 选择"用户行为"看板
   - 确认能正常加载和使用AI分析

## 数据文件处理

由于数据文件过大（300MB+），不能提交到Git：

**方案1：使用云存储**
- 将Excel文件上传到云存储（如AWS S3、阿里云OSS）
- 修改代码从云存储读取数据

**方案2：使用数据库**
- 使用Railway提供的PostgreSQL数据库
- 将数据导入数据库
- 修改代码从数据库读取

**方案3：定期手动上传**
- 通过Railway CLI上传数据文件
- 适合数据更新不频繁的情况
