# 部署步骤指南

## 📦 第一步：准备Git仓库

### 1. 打开PowerShell或CMD

在项目根目录（`E:\橙子学数据看板\橙子学数据看板`）右键，选择"在终端中打开"

### 2. 初始化Git仓库

```bash
git init
```

### 3. 添加所有文件

```bash
git add .
```

### 4. 提交

```bash
git commit -m "Initial commit: 数据看板"
```

## 🌐 第二步：创建GitHub仓库

### 1. 访问GitHub

打开浏览器，访问：https://github.com/new

### 2. 填写仓库信息

- **Repository name**: `data-dashboard`（或其他你喜欢的名字）
- **Description**: `数据看板`（可选）
- **Public/Private**: 选择 Public 或 Private
- ⚠️ **不要勾选**：
  - ❌ Add a README file
  - ❌ Add .gitignore
  - ❌ Choose a license

### 3. 点击"Create repository"

## 🔗 第三步：连接GitHub仓库

GitHub会显示一些命令，复制并执行：

```bash
# 添加远程仓库（替换成你的GitHub用户名和仓库名）
git remote add origin https://github.com/你的用户名/data-dashboard.git

# 设置主分支名称
git branch -M main

# 推送到GitHub
git push -u origin main
```

### 示例：

如果你的GitHub用户名是 `zhangsan`，仓库名是 `data-dashboard`，则命令是：

```bash
git remote add origin https://github.com/zhangsan/data-dashboard.git
git branch -M main
git push -u origin main
```

### 可能需要登录

第一次推送时，可能会弹出登录窗口，输入你的GitHub账号和密码（或使用Personal Access Token）。

## 🚀 第四步：部署到Vercel

### 1. 访问Vercel

打开浏览器，访问：https://vercel.com

### 2. 登录

点击右上角的 "Sign Up" 或 "Log In"

选择 "Continue with GitHub" 使用GitHub账号登录

### 3. 授权Vercel访问GitHub

首次登录时，需要授权Vercel访问你的GitHub仓库

点击 "Authorize Vercel"

### 4. 创建新项目

登录后，点击 "Add New..." → "Project"

### 5. 导入GitHub仓库

在列表中找到你刚才创建的 `data-dashboard` 仓库

点击 "Import"

### 6. 配置项目

在配置页面：

- **Project Name**: 保持默认或修改
- **Framework Preset**: 选择 "Other"
- **Root Directory**: 保持默认 `./`
- **Build Command**: 留空（不填）
- **Output Directory**: 留空（不填）
- **Install Command**: 留空（不填）

### 7. 点击"Deploy"

Vercel会开始部署，通常需要1-2分钟

### 8. 等待部署完成

部署完成后，会显示：
- ✅ 成功的标志
- 🌐 你的网站URL（例如：`https://data-dashboard-xxx.vercel.app`）

### 9. 访问你的网站

点击URL或"Visit"按钮，访问你的数据看板！

## 🔄 第六步：更新部署（以后使用）

当你修改代码后，想要更新线上版本：

```bash
# 1. 添加修改的文件
git add .

# 2. 提交修改
git commit -m "更新说明"

# 3. 推送到GitHub
git push
```

Vercel会自动检测到GitHub的更新，并自动重新部署！

## 🎨 第七步：自定义域名（可选）

如果你有自己的域名：

1. 在Vercel项目页面，点击 "Settings"
2. 点击 "Domains"
3. 输入你的域名
4. 按照提示配置DNS记录
5. 等待DNS生效（通常几分钟到几小时）

## 🐛 常见问题

### Q1: git push时要求输入用户名密码

**A**: GitHub已经不支持密码登录，需要使用Personal Access Token：

1. 访问：https://github.com/settings/tokens
2. 点击 "Generate new token" → "Generate new token (classic)"
3. 勾选 `repo` 权限
4. 点击 "Generate token"
5. 复制生成的token（只显示一次！）
6. 在git push时，用户名输入GitHub用户名，密码输入token

### Q2: Vercel部署失败

**A**: 查看部署日志：
1. 在Vercel项目页面，点击失败的部署
2. 查看 "Build Logs"
3. 找到错误信息
4. 根据错误信息修复问题

### Q3: 部署成功但页面显示404

**A**: 检查 `vercel.json` 配置是否正确，特别是routes部分。

### Q4: 数据看板显示空白

**A**: 
1. 按F12打开开发者工具
2. 查看Console标签的错误信息
3. 查看Network标签，看哪些文件加载失败
4. 检查文件路径是否正确
