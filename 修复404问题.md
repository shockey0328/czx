# 修复Vercel 404问题

## 🐛 问题原因

Vercel的配置文件过于复杂，导致路由无法正确工作。

## ✅ 已修复

已简化 `vercel.json` 配置文件，现在应该可以正常工作了。

## 🔄 重新部署步骤

### 方法1：通过Git推送（推荐）

```bash
# 1. 添加修改的文件
git add vercel.json

# 2. 提交修改
git commit -m "修复vercel.json配置"

# 3. 推送到GitHub
git push
```

Vercel会自动检测到更新并重新部署。

### 方法2：在Vercel控制台手动重新部署

1. 访问 https://vercel.com
2. 进入你的项目
3. 点击 "Deployments" 标签
4. 点击最新部署右侧的三个点 "..."
5. 选择 "Redeploy"
6. 点击 "Redeploy" 确认

## 🧪 测试

部署完成后，访问你的Vercel URL：
- 根路径：`https://your-project.vercel.app/`
- 应该自动跳转到dashboard.html

## 💡 如果还是404

### 检查1：确认dashboard.html在根目录

在项目根目录执行：
```bash
ls dashboard.html
# 或 Windows PowerShell:
Test-Path dashboard.html
```

应该返回 `True` 或显示文件存在。

### 检查2：查看Vercel部署日志

1. 在Vercel项目页面
2. 点击最新的部署
3. 查看 "Build Logs"
4. 看是否有错误信息

### 检查3：直接访问dashboard.html

尝试访问：
```
https://your-project.vercel.app/dashboard.html
```

如果这个能访问，说明文件存在，只是路由配置问题。

## 🔧 备用方案

如果上述方法都不行，可以尝试：

### 方案A：重命名dashboard.html为index.html

```bash
# 重命名文件
mv dashboard.html index.html

# 更新Git
git add .
git commit -m "重命名为index.html"
git push
```

Vercel默认会将index.html作为根路径的默认文件。

### 方案B：创建index.html跳转文件

创建一个简单的index.html：

```html
<!DOCTYPE html>
<html>
<head>
    <meta http-equiv="refresh" content="0;url=/dashboard.html">
    <title>跳转中...</title>
</head>
<body>
    <p>正在跳转...</p>
    <script>
        window.location.href = '/dashboard.html';
    </script>
</body>
</html>
```

## 📝 当前vercel.json配置

已简化为：

```json
{
  "version": 2,
  "routes": [
    {
      "src": "/",
      "dest": "/dashboard.html"
    },
    {
      "src": "/(.*)",
      "dest": "/$1"
    }
  ]
}
```

这个配置：
- 将根路径 `/` 重定向到 `/dashboard.html`
- 其他所有路径保持不变
